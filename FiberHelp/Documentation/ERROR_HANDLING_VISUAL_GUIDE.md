## Error Message Examples

### Database Errors

#### Duplicate Entry
```
User tries to create ticket with existing ID

Before: 
"System.Data.SqlClient.SqlException: Cannot insert duplicate key 
row in object 'dbo.Tickets' with unique index 'IX_Tickets_Id'. 
The duplicate key value is (123)."

After:
"?? Duplicate Entry: This ticket already exists in the system."
```

#### Foreign Key Violation
```
User tries to delete client with active tickets

Before:
"Microsoft.EntityFrameworkCore.DbUpdateException: The DELETE 
statement conflicted with the REFERENCE constraint 
"FK_Tickets_Clients_ClientId". The conflict occurred in database 
"FiberhelpDB", table "dbo.Tickets", column 'ClientId'."

After:
"?? Dependency Error: Cannot delete this client because it's 
linked to other records."
```

#### Connection Timeout
```
Database takes too long to respond

Before:
"System.Data.SqlClient.SqlException: Timeout expired. The timeout 
period elapsed prior to completion of the operation or the server 
is not responding."

After:
"?? Database connection timeout. Check your connection."
```

## Usage in Code

### Razor Page
```razor
@page "/example"
@inject ErrorHandlingService ErrorService

<!-- Add this at top of page -->
<ToastNotification />

<button @onclick="SaveData">Save</button>

@code {
    private async Task SaveData()
    {
        try
        {
            await DoSomething();
            ToastNotification.ShowSuccess("Data saved!");
        }
        catch (Exception ex)
        {
            ToastNotification.ShowException(ex, "Saving Data", ErrorService);
        }
    }
}
```

### Service Method
```csharp
public async Task<bool> CreateTicket(Ticket ticket)
{
    try
    {
        // Validation
        if (ticket == null)
        {
            LastError = "?? Ticket data is required";
            return false;
        }
        
        // Operation
        await _db.SaveAsync(ticket);
        return true;
    }
    catch (Exception ex)
    {
        LastError = _errorService.GetContextualMessage(
            ex, "create", "ticket"
        );
        return false;
    }
}
```

## Best Practices

### ? DO:
- Show toasts for all user actions
- Use appropriate toast types (success/error/warning/info)
- Provide context in error messages
- Keep messages short and clear
- Use icons for visual clarity
- Allow manual dismiss

### ? DON'T:
- Show technical error messages to users
- Stack too many toasts (max 3-4)
- Make toasts too long (5s is good)
- Block user interaction with modals for every error
- Show toasts for background operations
