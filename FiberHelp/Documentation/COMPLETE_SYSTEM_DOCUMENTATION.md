## Project Overview

**FiberHelp** is a Customer Relationship Management (CRM) system designed for fiber internet service providers. Built with **.NET MAUI Blazor Hybrid**, it provides:

- **Customer Information Management** - Track clients and accounts
- **Ticket Management** - Handle complaints and service requests
- **Billing & Accounting** - Invoice generation and expense tracking
- **Analytics & Reports** - Real-time dashboards and data visualization
- **Offline Support** - Work without internet, sync when connected

### Technology Stack

| Component | Technology |
|-----------|------------|
| Framework | .NET 9 MAUI Blazor Hybrid |
| Frontend | Blazor Components, CSS3 |
| Database (Cloud) | SQL Server (Azure-hosted) |
| Database (Local) | SQLite |
| ORM | Entity Framework Core |
| Authentication | Custom SHA256 password hashing |

## Installation Guide

### Prerequisites

- Visual Studio 2022 (17.8+) with .NET MAUI workload
- .NET 9 SDK
- SQL Server (LocalDB or Azure)

### Step 1: Clone Repository

```bash
git clone https://github.com/NeilJohn10/FInal_Project_FiberHelp.git
cd FiberHelp
```

### Step 2: Configure Database

1. Open `appsettings.json`
2. Update connection strings:

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "",
    "OnlineConnection": ""
  }
}
```

> **Note:** Connection strings and credentials are loaded from environment variables.
> See `.env.example` for required values.
```

### Step 3: Run Database Scripts

Execute in SQL Server Management Studio:

```sql
-- 1. Create tables
FiberHelp\Database\complete_setup.sql

-- 2. Add required columns
FiberHelp\Database\add_ticket_columns.sql
FiberHelp\Database\add_archive_columns_all.sql
FiberHelp\Database\create_technicians_table.sql

-- 3. Seed 100 records per table (REQUIRED for rubric)
FiberHelp\Database\seed_100_records.sql
```

### Step 4: Build and Run

```bash
dotnet build
dotnet run
```

### Default Login Credentials

Credentials are configured via environment variables (see `.env.example`):
- `FIBERHELP_ADMIN_EMAIL` — admin email address
- `FIBERHELP_ADMIN_PASSWORD` — admin password

Agents and technicians are created through the app’s Admin UI with secure PBKDF2 hashing.
No credentials are hardcoded in source code or SQL scripts.

## Core Features

### 1. Customer Information Management

**Location:** `/clients`

- **Add/Edit/Delete clients**
- **Search by name or email**
- **Archive/Restore functionality**
- **Link clients to accounts**
- **Export to CSV**

### 2. Ticket Management

**Location:** `/tickets`

- **Create tickets with priority levels** (Low, Medium, High, Critical)
- **Status workflow:** Open ? Assigned ? In Progress ? Resolved ? Closed
- **Assign technicians**
- **Archive old tickets**
- **Filter and search**

### 3. Billing & Accounting

**Location:** `/billing`

- **Create invoices** (Subscription, Service, One-Time)
- **Mark invoices as paid**
- **Track expenses by category**
- **Cash flow analysis**
- **Complete ledger view**
- **Aging report for receivables**

### 4. Analytics & Reports

**Location:** `/` (Dashboard) and `/reports`

- **Real-time KPIs:** Open tickets, Revenue, Resolution rate
- **Interactive charts:** Ticket trends, Revenue vs Expenses, Priority distribution
- **Client growth tracking**
- **Export reports to CSV**

## Offline/Online Synchronization

### How It Works

```
???????????????????????????????????????????????????????????????
?                  SYNC WORKFLOW                              ?
???????????????????????????????????????????????????????????????
?                                                             ?
?  1. USER CREATES/UPDATES RECORD                             ?
?     ?                                                       ?
?     ?                                                       ?
?  ???????????????????????????????????????????????????????   ?
?  ?  DualWriteService.EnqueueAsync()                    ?   ?
?  ?  - Writes to LOCAL SQLite immediately               ?   ?
?  ?  - Adds message to OutboxMessages table             ?   ?
?  ???????????????????????????????????????????????????????   ?
?     ?                                                       ?
?     ?                                                       ?
?  2. OUTBOX PROCESSOR (Background, every 10 seconds)         ?
?     ?                                                       ?
?     ?                                                       ?
?  ???????????????????????????????????????????????????????   ?
?  ?  OutboxProcessor.RunAsync()                         ?   ?
?  ?  - Checks if cloud is reachable                     ?   ?
?  ?  - Processes pending messages in order              ?   ?
?  ?  - Applies changes to CLOUD SQL Server              ?   ?
?  ?  - Marks messages as processed                      ?   ?
?  ???????????????????????????????????????????????????????   ?
?     ?                                                       ?
?     ?                                                       ?
?  3. DATA SYNC (Periodic full sync)                          ?
?     ?                                                       ?
?     ?                                                       ?
?  ???????????????????????????????????????????????????????   ?
?  ?  DataSyncService.SyncCloudToLocalAsync()            ?   ?
?  ?  - Pulls all data from CLOUD                        ?   ?
?  ?  - Refreshes LOCAL cache                            ?   ?
?  ???????????????????????????????????????????????????????   ?
?                                                             ?
???????????????????????????????????????????????????????????????
```

### Key Components

| Component | Purpose |
|-----------|---------|
| `DualWriteService` | Writes locally + queues for cloud sync |
| `OutboxProcessor` | Processes queued messages to cloud |
| `DataSyncService` | Full sync from cloud to local |
| `SyncScheduler` | Schedules periodic sync operations |

## Security Features

### Authentication

- **PBKDF2 password hashing** (100,000 iterations, random salt, constant-time comparison)
- **Legacy SHA256 auto-upgrade** on login
- **Account lockout** after 5 failed attempts (progressive suspension)
- **Role-based access control** (Administrator, Agent, Supervisor, Technician)
- **Session management** via AuthService
- **CAPTCHA verification** on login

### Password Requirements

- Minimum 8 characters
- At least one uppercase letter
- At least one digit
- At least one special character

### Data Protection

- **HTTPS** for cloud connections
- **Parameterized queries** via EF Core (SQL injection protection)
- **Input validation** on all forms
- **AES-256 encryption** for sensitive data at rest
- **Environment variables** for all sensitive configuration (no hardcoded credentials)
- **SQL data masking** for PII in database views

## Troubleshooting

### Common Issues

| Issue | Solution |
|-------|----------|
| "Invalid password" on correct credentials | Check password hash matches; run DbInitializer |
| Cloud sync not working | Verify OnlineConnection string in appsettings.json |
| Tables not found | Run complete_setup.sql and migration scripts |
| Technicians not showing | Run create_technicians_table.sql |

### Debug Logs

Check Visual Studio Output window for:
- `AuthService:` - Login attempts
- `AdminService:` - CRUD operations
- `DualWriteService:` - Sync operations
- `OutboxProcessor:` - Cloud sync status
