# FiberHelp Technician Role System

## Overview

This document describes the Technician role implementation in FiberHelp, enabling field technicians to receive ticket assignments from agents and resolve them.

## Database Schema Changes

### Agents Table (New Columns)

| Column | Type | Default | Description |
|--------|------|---------|-------------|
| Role | NVARCHAR(50) | 'Support Agent' | Role: Support Agent, Technician, Supervisor |
| ServiceArea | NVARCHAR(200) | NULL | Geographic area for technicians |

### Tickets Table (New Columns)

| Column | Type | Description |
|--------|------|-------------|
| AssignedAgentId | NVARCHAR(50) | Agent who assigned the ticket |
| AssignedTechnicianId | NVARCHAR(50) | Technician assigned to resolve |
| ResolvedByTechnicianId | NVARCHAR(50) | Technician who resolved |
| AssignedAt | DATETIME2 | When ticket was assigned |
| ResolvedAt | DATETIME2 | When ticket was resolved |
| ResolutionNotes | NVARCHAR(1000) | Technician's resolution notes |

## New Pages

### Technician Dashboard (`/technician`)

- Shows only tickets assigned to the logged-in technician
- Stats: Pending, In Progress, Resolved Today
- Actions: Start Work, Resolve
- Resolution modal with notes input

## Authentication Changes

The `AuthService` now supports role-based access:

```csharp
// Role helper properties
AuthService.IsAdministrator  // true if Administrator
AuthService.IsAgent          // true if Agent or Support Agent
AuthService.IsTechnician     // true if Technician
AuthService.IsSupervisor     // true if Supervisor
AuthService.CanManageTickets // Admin, Agent, or Supervisor
AuthService.CanResolveTickets // Technician or Admin
AuthService.CanAssignTechnicians // Agent, Admin, or Supervisor
```

## How to Create a Technician

1. Log in as Administrator
2. Go to **Agents & Techs** page
3. Click **+ Register**
4. Select **Role: Technician**
5. Fill in:
   - Email
   - Full Name
   - Department (e.g., "Field Operations")
   - Service Area (e.g., "Metro Manila")
   - Phone
   - Password
6. Click **Create**

## API Methods

### AdminService

```csharp
// Get all technicians
Task<List<Agent>> GetTechniciansAsync()

// Get available (active, not locked) technicians
Task<List<Agent>> GetAvailableTechniciansAsync()

// Assign technician to ticket
Task<bool> AssignTechnicianToTicketAsync(int ticketId, string technicianId, string assignedByAgentId)

// Resolve ticket by technician
Task<bool> ResolveTicketByTechnicianAsync(int ticketId, string technicianId, string? resolutionNotes)

// Get tickets for a specific technician
Task<List<Ticket>> GetTicketsForTechnicianAsync(string technicianId)

// Get pending tickets for technician
Task<List<Ticket>> GetPendingTicketsForTechnicianAsync(string technicianId)

// Get count of active tickets for technician
Task<int> GetTechnicianTicketCountAsync(string technicianId)
```
